在Go语言中通过go关键字并发执行任务并不是开启了多个线程，而是使用的协程（coroutine），对于计算机来说同时开启1000个线程压力还是很大的

协程：

  • 轻量级“线程”
  • 非抢占式多任务处理，由协程主动交出控制权
  • 编译器/解释器/虚拟机层面的多任务
  • 多个协程可能在一个线程或多个线程上运行

Go语言中的协程叫goroutine，Go语言很特殊main函数也是一个goroutine

import (
	"fmt"
	"runtime"
	"time"
)

func main() {
	var a [10]int
	for i := 0; i < 10; i++ {
		go func(i int) {
			for {
				a[i]++
				runtime.Gosched()
			}
		}(i)
	}
	time.Sleep(time.Millisecond)
	fmt.Println(a)
}

在上面的程序中有一个问题需要注意，我们的程序如果使用函数式编程的思想，那么并发的匿名函数是不需要传递i的值的，但是这里为什么需要传递呢？当跳出外层for循环时，并发的goroutine并没有结束，但是这时候i是等于10的，所以去访问a[10]的时候会抛出异常

但是上面的程序还有一个问题，在执行fmt.Println(a)的时候，如果a还在写入，那么就还是会发生冲突，那么这时候就需要使用channel来解决了。

Coroutine

一个普通的函数就是运行在一个线程上的，但是协程可能运行在一个线程上，也可能运行在多个线程上。

在Go语言中goroutine怎么去开启，一个线程上运行几个goroutine都不需要我们去管，只需要使用go关键字交给调度器就行了，其他的事情调度器会帮我们完成

goroutine和传统意义上的协程有一些区别：

  • 任何函数只需要加上go关键字就能给调度器运行
  • 不需要在定义函数时区分是不是异步函数
  • 调度器会在合适的点进行切换

goroutine可能切换的点：

  • I/O，select
  • channel
  • 等待锁
  • 函数调用（有时）
  • runtime.Gosched()

就算启动一万个goroutine，但是go语言最多会启动计算机核心数个线程

func recv(ch chan int) {
	for {
		n := <-ch
		fmt.Println(n)
	}
}

func main() {
	ch := make(chan int)
	go recv(ch)
	ch <- 1
	ch <- 2
	ch <- 3
}

func worker(id int, ch chan int) {
	for {
		fmt.Printf("worker %d received %c \n", id, <-ch)
	}
}

func main() {
	var channles [10]chan int
	for i := 0; i < 10; i++ {
		channles[i] = make(chan int)
		go worker(i, channles[i])
	}

	for i := 0; i < 10; i++ {
		channles[i] <- 'a' + i
	}
	time.Sleep(time.Millisecond)
}

但是输出的顺序可能是乱序的，因为输出是一个IO操作

func createWorker(id int) (ch chan<- int) {
	ch = make(chan int)
	go func() {
		for {
			fmt.Printf("worker %d received %c \n", id, <-ch)
		}
	}()
	return ch
}

func main() {
	var channles [10]chan<- int
	for i := 0; i < 10; i++ {
		channles[i] = createWorker(i)
	}

	for i := 0; i < 10; i++ {
		channles[i] <- 'a' + i
	}
	for i := 0; i < 10; i++ {
		channles[i] <- 'A' + i
	}
	time.Sleep(time.Millisecond)
}

func main() {
	ch := make(chan int, 3)
	go worker(ch)
    ch <- 1
	ch <- 2
	ch <- 3
    close(ch)
	time.Sleep(time.Millisecond)
}

执行这个程序之后发现关闭这个channel之后竟然还在一直收数据，而且全是0，因为这个时间点收数据的goroutine还不知道我们已经关闭了channel，所以会一直收到类型的零值，所以在收数据的时候我们需要判断一下是否收到数据，那么worker需要做如下更改

func worker(ch chan int) {
	for {
		n, ok := <-ch
		if !ok {
			break
		}
		fmt.Println("received", n)
	}
}
