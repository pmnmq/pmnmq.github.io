poetry - python 的包管理工具

以下功能中不管是静态检查插件、格式化工具配置、launch profile 等，基本都可以使用 pycharm 和 vscode 做到，但是都同时做到是不可能的，像 black 在 pycharm 中配置起来还是比较麻烦，开发环境依赖都需要借助第三方包管理工具，并且依赖关系混乱，launch profile 每一个编辑器都要配置，而 poetry 同时做到了，并且让所有端的体验都相同

前情提要：阅读本文之前你至少需要知道什么是虚拟环境，笔者使用操作系统为 Debian，编辑器为 Neovim

安装

1

pip3 install poetry

设置源

方式一

使用 poetry-plugin-pypi-mirror 插件实现全局替换

  1. 安装插件
1

poetry self add poetry-plugin-pypi-mirror

使用 poetry 安装插件速度可能会很慢，可以使用 pip 进行安装

1

pip3 install poetry-plugin-pypi-mirror
  1. 配置

修改poetry配置文件 $HOME/.config/pypoetry/config.toml

1
2
3

[plugins]
[plugins.pypi_mirror]
url = "https://pypi.tuna.tsinghua.edu.cn/simple/"

方式二

项目创建后在项目配置文件中配置

1
2
3
4

[[tool.poetry.source]]
name = "tsinghua"
default = true
url = "https://pypi.tuna.tsinghua.edu.cn/simple/"

初始化

  1. 新建项目
1

poetry new project-name
  1. 已有项目初始化
1

poetry init

配置 mypy 插件(如果需要)

mypy 插件的作用是帮助 python 做静态类型检查

  1. sqlalchemy-stubs

    sqlalchemy-stubs -> sqlalchemy

    安装

    1

    poerty add --group=dev sqlalchemy-stubs

    1
2

    [tool.mypy]
plugins = ["sqlmypy"]

  2. pydantic.mypy

    pydantic.mypy -> pydantic

    1
2

    [tool.mypy]
plugins = ["pydantic.mypy"]

配置格式化工具

编辑 pyproject.toml

1
2
3
4
5
6
7
8
9

[tool.black]
line-length = 100
target-version = ['py38', 'py39']

[tool.isort]
profile = "black"
include_trailing_comma = true
multi_line_output = 3
skip = [".venv", ".git"]

安装isort和black为项目开发依赖

1

poetry add --group=dev isort black

安装 taskipy 辅助工具

taskipy 能够帮助使用poetry的开发者简化命令

1

poetry add --group=dev taskipy

当我们需要使用测试框架或者运行代码有复杂的启动参数的时候，taskipy可以帮助我们简化这个过程，以 isort 和 black 举例

1
2
3

[tool.taskipy.tasks]
isort = "isort ."
black = "black ."

这里参数少是因为在上面已经配置过了，所以这里需要写的参数很少

查看已有 task

1

poetry run task --list

运行 task

1

poetry run task black

安装第三方包

1

poetry add package-name

poetry 中如果不同环境有不同的依赖可以使用 group 进行区分，比如开发环境、文件构建环境、测试环境、生成环境等

通过指定 group 参数对依赖进行分组

1

poetry add --group=dev isort

当对一个已有的 poetry 项目安装依赖的时候可以指定只安装的分组

1

poetry install --only dev

因为场景过于丰富，超过了笔者使用范畴，更多使用方法查看官方文档 https://python-poetry.org/docs/managing-dependencies/#optional-groups

一些可能有用的配置

$HOME/.config/pypoetry/config.toml

1
2

[virtualenvs]
in-project = true  # 将虚拟环境创建在项目中

结语

使用集成环境固然方便，但是当面临昂贵的许可证的时候我选择了开源的 Neovim ，因为vscode在 python 上的体验并不好，或者说是在动态类型语言上的体验都不好。还有一个原因，当项目上线的时候如果不知道项目怎么做环境隔离那么这个项目是非常危险的，除了写代码还应该掌握更多周边的技能
