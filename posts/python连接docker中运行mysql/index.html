因为作业的原因我需要使用MySQL5.7来存储数据，但是在AUR上只能安装mariadb。我也尝试通过MySQL官网打包的通用安装包安装，但是一直失败，于是只能通过docker来运行MySQL了。

安装docker

  1. 安装
1

sudo pacman -S docker
  1. 启动
1

systenctl start docker
  1. 将当前用户添加到docker用户组
1

sudo gpasswd -a pujic docker

  1. 退出当前用户重新登录

  2. 运行hello-world

1

docker run hello-world

创建MySQL容器

  1. 拉取MySQL5.7镜像（速度太慢可以配置阿里云镜像地址）
1

docker pull mysql:5.7
  1. 创建MySQL配置文件
1
2

sudo mkdir -p /opt/mysql/{conf, data}
sudo nvim /opt/mysql/conf/my.cnf

my.cnf添加如下内容

1
2
3
4
5
6
7
8

[mysqld]
user=mysql
character-set-server=utf8
default_authentication_plugin=mysql_native_password
[client]
default-character-set=utf8
[mysql]
default-character-set=utf8
  1. 创建MySQL5.7容器
1

docker run -d -p 3306:3306 --privileged=true -v /opt/mysql/conf/my.cnf:/etc/mysql/my.cnf -v /opt/mysql/data:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=p --name mysql_container mysql:5.7

参数说明

参数解释
-p端口映射，冒号前面的端口是向外暴漏的端口
–privileged=true容器内的root拥有真正root权限，否则容器内root只是外部普通用户权限
-v文件和文件夹映射
-e MYSQL_ROOT_PASSWORD设置root用户的密码
–name mysql_container容器的名称，可以直接通过容器名称关闭启动容器

配置MySQL

  1. 进入容器内的命令行
1

docker exec -t mysql_container bash
  1. 登录MySQL
1

mysql -u root -p
  1. 创建MySQL用户
1

CREATE USER 'pujic'@'%' IDENTIFIED BY 'a';
  1. 对用户进行授权
1

GRANT ALL ON *.* TO 'pujic'@'%';

Python连接测试

import pymysql

conn = pymysql.Connection(host='localhost', port=3306, user='pujic', password='a')
cursor = conn.sursor()
cursor.execute('SELECT VERSION()')
print(cursor,fetchone())
